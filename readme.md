# MAX rent bot v1

Современный бот для MAX, который показывает свободные помещения АО «ТомскАгроИнвест», открывает карточки объектов и подводит пользователя к заявке.

## Возможности

- приветствие и главное меню;
- меню команд MAX: `/start`, `/menu`, `/clear`, `/search`, `/archive`, `/help`;
- список свободных помещений из `data/premises.csv` сразу кнопками, с пагинацией по 5 объектов;
- отдельные CSV-каталоги для «Свободные торговые места» (`data/trade_places.csv`) и «Специальные предложения» (`data/special_offers.csv`);
- карточка помещения с площадью, ценой, типом, зданием, этажом и описанием;
- быстрый возврат к списку;
- команда `/clear` для полного удаления доступной боту истории сообщений в чате и показа главного меню;
- команда `/search` для поиска по площади, стоимости за м² и типу помещения;
- команда `/archive` для просмотра ранее поданных пользователем заявок;
- форма обратной связи по кнопке «Подать заявку» с сохранением в `data/applications.csv`;
- файлы-заглушки документов в `docs/`;
- заглушки для документов и контактов, чтобы меню v1 было цельным.

## Запуск

1. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

2. Создайте `.env` и укажите токен:

   ```env
   BOT_TOKEN=your_max_bot_token
   ```

3. Запустите бота:

   ```bash
   python app.py
   ```

## Поиск

Команда `/search` принимает параметры в формате `ключ=значение`:

```bash
/search area=50 price_m2=900 type=склад
```

Поддерживаемые параметры:

- `area` или `площадь` — максимальная площадь в м²;
- `price_m2` или `цена_м2` — максимальная стоимость за м²;
- `type` или `тип` — тип помещения: `склад`, `павильон`, `офисное`, `торговое`.

Примеры:

```bash
/search type=склад
/search area=30 type=павильон
/search price_m2=800 type=офисное
```

## Данные

Основной CSV-файл помещений находится в `data/premises.csv`. Для публикации помещения в основном каталоге установите `status` в `free`. Поле `special_offer=yes` в этом файле не переносит объект в отдельный раздел, а включает более яркое оформление карточки и выделение в списке.

Отдельные разделы используют отдельные CSV:

- `data/trade_places.csv` — поля `tiitle`, `building`, `price`, `description`;
- `data/special_offers.csv` — поля `tiitle`, `price`, `description`;
- `data/applications.csv` — создаётся автоматически при первой заявке, поля `data`, `user_id`, `area`, `price`, `description`; `user_id` нужен для команды `/archive`.

## Упрощение структуры

Удалены лишние файлы, которые были пустыми, дублировали будущую функциональность или не использовались в v1:

- `Bot.zip` — архив исходников больше не нужен после распаковки проекта в репозиторий;
- `handlers/application.py`, `handlers/contacts.py`, `handlers/documents.py` — пустые обработчики заменены callback-логикой в `handlers/callbacks.py`;
- `handlers/premises.py` — отдельный обработчик каталога заменён единым `handlers/messages.py`, чтобы команды `/menu`, `/clear`, `/help` и текстовые пункты меню не конфликтовали;
- `keyboards/keyboards.py` — пустой файл, общий builder находится в `keyboards/main_menu.py`;
- `models/application.py` — модель заявки не используется до полноценного сценария заявок;
- `repository/repository.py` — абстракция не используется текущим CSV-репозиторием;
- `services/document_service.py`, `services/menu_service.py`, `services/pagination.py` — пустые/неиспользуемые сервисы;
- `storage/session.py` — in-memory session не используется текущими сценариями;
- `utils/csv_utils.py`, `utils/validators.py` — пустые утилиты.
